Release 3.0.5 - 02 Jul 2026
---------------------------
* Performance: neighbours now appear in the results box the instant each one is
  decoded instead of only after the full 60-second listen window closes. The
  "show all neighbours" window still runs its course to catch additional
  switches, but a single-neighbour result (the common case) now feels instant
  again rather than always taking a full minute
* Status bar shows "Found N neighbour(s) - still listening for more..." while
  the capture continues, and Stop can be used to end early once enough is shown


Release 3.0.4 - 24 Jun 2026
---------------------------
* Show all neighbours: LDWin now listens for the full 60-second capture window
  and collects every distinct CDP/LLDP neighbour that announces itself on the
  link (deduplicated by protocol + device + port). Each neighbour is displayed
  with a "Neighbour 1 (CDP) — Switch-A", "Neighbour 2 (LLDP) — Switch-B" header
  so it is clear which announcement came from which device.
* Status bar now summarises the number of neighbours found ("Found N neighbours.")
* LLDP 802.1 VLAN Name / Port VLAN ID, CDP VTP domain and Duplex TLVs are now
  decoded and displayed in both the main report and the raw-TLV view.

Release 3.0.2 - 24 Jun 2026
---------------------------
* Distribution change to reduce antivirus false positives: releases now ship a zipped
  self-contained folder (LDWin-win-x64.zip with LDWin.exe + its DLLs) instead of a
  compressed single-file exe, avoiding the self-extraction-to-%TEMP% pattern that some
  AV heuristics flag. Extract the zip and run LDWin.exe. No functional changes.

Release 3.0 - 24 Jun 2026
-------------------------
* Rewritten in C# / .NET 8 (WinForms). CDP and LLDP are now captured and decoded
  in-process via SharpPcap/Npcap - there is no bundled tcpdump.exe to ship or keep
  alongside the program, and nothing is extracted to %TEMP%
* Ships as a single self-contained LDWin.exe
* Solution split into LDWin.Core (capture + decoders, net8.0, no UI), LDWin (WinForms
  GUI), and LDWin.Tests (xUnit unit tests for the CDP/LLDP decoders)
* Decoder unit tests run on every push/PR via the Test workflow
* The legacy AutoIt implementation (LDWin.au3) and its tcpdump-based build have been
  removed; see release 2.3 and earlier for that history
* Carried over from the 2.x Windows 11 work: capture works with "Core Isolation >
  Memory Integrity" (HVCI) enabled, since Npcap's driver is signed for and compatible
  with Core Isolation (it does not need to be disabled)


Release 2.3 - 23 Jun 2026
-------------------------
* Windows 11 compatibility: link discovery now works with "Core Isolation >
  Memory Integrity" (HVCI) enabled
* Capture now uses Npcap instead of the old Microolap/WinPcap-era driver that
  HVCI blocks. Npcap's driver is signed for and compatible with Core Isolation
* LDWin now detects whether Npcap is installed and, if not, points the user to
  https://npcap.com/ to install it (Core Isolation does not need to be disabled)
* Adapter is now opened via the interface tcpdump reports for the GUID (-D),
  falling back to Npcap's \Device\NPF_{GUID} naming
* Fixed a crash ("Subscript used on non-accessible variable") when selecting a
  network card whose details were incomplete - values are now read by key
* tcpdump capture errors (bad interface, missing wpcap.dll, ...) are now shown
  instead of silently reporting "no link data found"
* tcpdump.exe now ships next to LDWin.exe and runs in place - it is no longer
  extracted to and executed from %TEMP% (a pattern antivirus tools flag). Keep
  the two files in the same folder
* Build stamps version/publisher metadata and no longer compresses the exe, to
  reduce antivirus false positives
* BUILD NOTE: the bundled tcpdump.exe is built against Npcap's wpcap.dll by the
  GitHub Actions workflow (.github/workflows/build.yml), not the old self-
  contained Microolap build
